What I'm running right now

Trust is the infrastructure. Here are the tools, models, and hosting I'm running this quarter, what I'm building next, and what I've killed along the way. Version numbers and model names included.

Last updated: July 2026

// What I'm running

The live stack

Everything below is running in production for Evalve today, most of it self-hosted on my own hardware. If you hire me, this is the caliber of build you get, not slideware.

Assistant & orchestrationClaude Code
An always-on executive assistant built on Claude Code, routing work across Opus 4.8, Sonnet, and Haiku 4.5 so the simple jobs run on the cheapest model that can do them. Runs 24/7 on a Mac Studio M4 Pro at home.
Workflow automationn8n · self-hosted
A self-hosted n8n instance (Docker, on the Mac Studio) runs the Evalve automations: discovery follow-ups, newsletter hand-off, lead-magnet tracking, welcome replies, and error alerts. No Zapier, no per-task SaaS fees.
Local AI inferenceQwen 3 · tinygrad
Open-source models (Qwen 3) served on tinygrad over a Mac Studio M4 Max with an external GPU, with automatic fallback to cloud models when a job needs more power. Sensitive data never leaves the network.
Assistant commsMatrix / Synapse
A self-hosted Matrix homeserver (Synapse), end-to-end encrypted, is becoming the primary channel I talk to the assistant through, read in Element X. Migrating off Telegram, which stays as a fallback.
Ops dashboardNext.js · Vercel
The Evalve Dashboard (Next.js on Vercel, Supabase Postgres) is the CRM and command center: outreach pipeline, morning briefing, email sync, and calendar, with a test suite kept green.
Newsletter, blog & communityBeehiiv
Evalve Out Loud runs on Beehiiv: the newsletter, a gated blog, and a free community at newsletter.evalveconsulting.com, on a custom sending domain.
This websiteGitHub Pages
evalveconsulting.com is a hand-built static site on GitHub Pages, no CMS. Every page is version-controlled and deploys on push. (This page included.)
Social publishingBuffer
Buffer schedules to LinkedIn (primary) and Instagram (secondary). The weekly AI Tip Card is rendered from a brand-locked HTML template into a 1080x1350 image.
Podcast productionDescript
Evalve Out Loud is edited in Descript (edit-by-text), recorded on a multicam iPhone rig with a Rode wireless split and a two-track Logic capture for in-person episodes.
Email & calendarGoogle · MCP
Gmail and Google Calendar wired into the assistant over MCP for triage and scheduling, with Fantastical as the natural-language calendar front end.
Payments & bankingStripe · Mercury
Stripe handles product checkout and the working-session booking; Mercury is the business bank.

In the workshop this quarter

Apps

On TestFlight · iterating

Social media automations

// What I killed

The graveyard

Killed tools, left visible on purpose. A clean list hides the learning; the graveyard is where the real decisions show. Each one has a date and a one-line reason.

jarvis-vps + OpenClawKilled 2026-06-03
Retired the cloud VPS and consolidated the whole assistant stack onto hardware I own. Cheaper, more private, one less thing to babysit.
Telegram as primary commsRetiring 2026-07
Moving assistant comms to self-hosted Matrix for ownership and control. Telegram stays as a fallback channel, not the front door.
Daily cold-outreach engineDemoted 2026-07-06
It ran cheap but converted nothing. Districts hire on referrals, credibility, and case studies, so personal energy moved to content and warm relationships.
Gumroad for productsKilled 2026-07-03
Consolidated product sales onto Beehiiv so the store lives where the audience already is, instead of a second platform to maintain.
Five-platform social spreadKilled 2026-07-03
Trying to be everywhere diluted everything. Focused on LinkedIn for text and Instagram for video, with auto-crossposting to the rest for free.
Duplicate morning briefingsKilled 2026-07-08
Two automations were posting overlapping daily digests. Folded them into a single 6am briefing so the day starts with one message, not three.
